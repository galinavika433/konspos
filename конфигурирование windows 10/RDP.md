**Протокол:** Microsoft для удаленного доступа к Windows
Порт **3389**. Шифрование, редирект устройств

### Последовательность подключения

1. **Инициация** — обмен X.224, согласование метода безопасности (Standard RDP, TLS, CredSSP/NLA)
2. **Обмен базовыми настройками** — MCS Connect Initial/Response, передача параметров клиента (размер экрана, раскладка)
3. **Соединение каналов** — присоединение пользовательского канала и виртуальных каналов
4. **Начало безопасности** — обмен ключами (для Standard RDP — клиент шифрует свой nonce публичным ключом сервера)
5. **Обмен защищенными настройками** — Client Info PDU (имя пользователя, пароль для автовхода)
6. **Лицензирование** — передача лицензии от сервера клиенту
7. **Обмен возможностями** — Demand Active → Confirm Active (согласование функций)
8. **Финализация** — синхронизация, управление, список шрифтов; после этого начинается обмен графикой и вводом

### mstsc подключение

```
mstsc [/v:<сервер[:порт]>] [/admin] [/f] [/w:<ширина> /h:<высота>]
```


```
mstsc /v:192.168.1.100       # по IP
mstsc /v:server:3390         # другой порт
mstsc /f                     # полный экран
mstsc /admin                 # консоль сервера
```
