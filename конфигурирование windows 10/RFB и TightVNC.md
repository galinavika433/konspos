RFB- это сетевой протокол для доступа к графическому интерфейсу пользователя. Он работает на уровне кадрового буфера (framebuffer)

- **RFB сервер** - сторона, где происходят изменения в кадровом буфере (оконная система и приложения)

- **RFB клиент** - сторона пользователя, отображающая графический интерфейс и принимающая ввод с клавиатуры/мыши

### Этапы работы

1. **Рукопожатие**

- Согласование версии (3.3, 3.7, 3.8)

- Выбор метода безопасности (None, VNC Auth, Tight)

2. **Инициализация**
- Клиент → `ClientInit`

- Сервер → `ServerInit` (размеры, формат пикселей, имя рабочего стола)

3. **Основной цикл**

- Клиент запрашивает обновления (`FramebufferUpdateRequest`)

- Сервер отправляет изменения (`FramebufferUpdate`) — только изменившиеся прямоугольники

- Клиент асинхронно передает события мыши/клавиатуры


### Подключение через TightVNC

**Клиент (vncviewer):**

```
vncviewer host:display          # display = номер (порт 5900+display)
vncviewer host::port            # явный порт
vncviewer -via gateway host     # через SSH-туннель
vncviewer -listen               # режим ожидания (обратное подключение)
```

**Сервер (vncserver):**

```
vncserver -localhost            # только локальные подключения (рекомендуется)
vncpasswd                       # установка пароля VNC
```

**Типовой безопасный вариант (через SSH):**

1. На сервере: `vncserver -localhost`

2. На клиенте: SSH-туннель (локальный порт → сервер:590X)

3. Подключение: `vncviewer localhost:590X`
