# Как создать контейнер?

**Контейнер в Docker Desktop создаётся на основе образа** — «шаблона», внутри которого работает приложение. Можно использовать готовые образы из хранилища Docker Hub или создать собственный образ с помощью Dockerfile — текстового файла, содержащего инструкции по сборке окружения.

**Пошаговая инструкция**

**Загрузка и запуск контейнера из Docker Hub**:

- Открыть Docker Desktop и перейти на «Панель управления».
- Использовать панель поиска, чтобы найти образ (например, официальный образ Nginx).
- Нажать на образ, чтобы скачать его на локальную машину.
- Нажать кнопку «Запустить».
- После запуска контейнера открыть веб-браузер и перейти по адресу, например — должна появиться страница приветствия Nginx.



**Создание собственного образа с помощью Dockerfile**: 

- Создать Dockerfile в папке, где лежит приложение. В файле указать инструкции по сборке окружения: установку зависимостей, копирование файлов проекта, настройку переменных среды и запуск приложения.



- **Собрать образ** в терминале в папке с Dockerfile. Например, выполнить команду: 

 ```python
 docker build -t my-python-app
 ```
Флаг **-t** задаёт имя образа, а символ точки в конце — путь к Dockerfile (текущая директория).



- **Запустить контейнер** на основе созданного образа. Например, использовать команду: 
 ```python
 docker run --name my-python-container -p 8080:8080 my-python-app

 ```
Параметр **-p** пробрасывает порт 8080 контейнера на порт 8080 на хост-машине.


# Как создать образ?

**Пример содержимого Dockerfile для Python-приложения с Nginx:**

```python
# Базовый образ
FROM python:3.9-slim

# Установка зависимостей
RUN apt-get update && apt-get install -y nginx

# Копирование файлов проекта
WORKDIR /app
COPY . .

# Установка Python-зависимостей
RUN pip install -r requirements.txt

# Настройка переменных окружения
ENV FLASK_APP=app.py

# Открытие порта
EXPOSE 80

# Команда для запуска приложения
CMD ["gunicorn", "--bind", "0.0.0.0:80", "app:app"]

```

## Сборка образа

Для сборки образа используется команда `docker build .`
**Базовая команда:**
```puthon
docker build .
```


**С указанием тега (имени образа):**
```python
docker build -t my-username/my-image .
```
Здесь  `-t` или `--tag` задаёт имя образа и тег. Часть имени до символа` / `должна совпадать с именем вашего пользователя Docker Hub. 

**Точка в конце команды** означает, что Dockerfile находится в текущей директории, из которой выполняется сборка. 
## Проверка созданного образа

После сборки можно проверить список образов с помощью команды:
```python
docker image ls
```

В выводе будет отображаться информация об созданном образе, его теге, ID, дате создания и размере.
