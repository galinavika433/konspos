Пуш - это команда в системах контроля версий Git, которая отправляет локальные коммиты на удалённый репозиторий (например, GitHub, GitLab). Это процесс «загрузки» изменений, сделанных у вас локально, чтобы они стали доступны другим разработчикам и служили актуальной версией проекта на сервере. 

**Зачем нужен пуш** - Делать локальные изменения видимыми для команды - Синхронизировать работу с удалённым репозиторием - Обновлять общедоступную историю проекта - Начинать процесс слияния (merge) и запросов на изменения (pull requests)
